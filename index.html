    Realizei as configurações no ambiente Linux Kubuntu 14.04.

    # Vídeo I – Configuração básica do ambiente.

    • Conferir a versão do python instalada, no meu caso a versão já instalada é 2.7.6 (comando: python –version).
    • Instalação do virtualenv (comando: apt-get install python-virtualenv).

    ## Vídeo II – Projeto tekton.

    • Realizado o download do appengine diretamente do fabricante (google) , e descompactado o arquivo .tar.gz na pasta de minha escolha /home/programas/ .
    • Configurado o arquivo .bashrc para que sempre a variável de ambiente configure o path do appengine. No meu caso foi necessário incluir a linha: export PATH=/home/rogerio/programas/google_appengine:$PATH . Para o path funcionar corretamente foi necessário reiniciar o todo ambiente.
    • Testado a configuração do path corretamente após a reinicialização do ambiente.
    • Com boa parte do ambiente configurado, foi realizado o download do projeto tekton.
    • Após o download, foi necessário instalar as bibliotecas necessárias para rodar o projeto tekton. Para isso foi executado o arquivo venv.sh .
    • Com as bibliotecas instaladas foi realizado um teste no meu servidor local (localhost:8080), abrindo a página inicial do projeto tekton pelo browser.

    ## Vídeo III – Pycharm

    • Realizado o download da IDE pycharm pela página do próprio fabricante. Uma das vantagens de se utilizar uma IDE é a opção de DEBUGAR o código fonte.
    • Com o pycharm instalado, foi aberto o projeto TEKTON .
    • Configurado o interpretador do projeto juntamente com o ambiente virtual já criado anteriormente.
    • Setada as pastas apps, appengine e test, indicando que são pastas de códigos fontes
    • Compilado o código fonte e executado o projeto, abrindo-o no servidor local localhost:8080.

    Respostas dos capítulos I, II e III .

    ## Capítulo I 1. Quais as duas principais versões atuais do interpretador Python? As duas principais versões atuais do interpretador Python são 2.7 e 3.4.

    1. Qual versão de Python é suportada atualmente pelo App Engine? A versão suportada atualmente é a versão 2.7. Com esta versão possui várias atualizações, como por exemplo 2.7.1 , 2.7.2 por assim adiante, todas essas versões podem ser utilizadas.

    2. Para que serve o SDK do App Engine? O sdk do appengine fornece a plataforma necessária para que o desenvolvedor hospede seu código web na plataforma google, facilitando sua própria vida sem se preocupar com a configuração de servidores (como por exemplo a amazon), ou com a configuração de banco de dados relacionais (como por exemplo mysql). Esse plataforma fornecida pode ser utilizada com as lnguagens: java e python.

    3. Qual a vantagem de colocar ferramentas instaladas no path do Sistema Operacional? A grande vantagem é poder executar qualquer comando no terminal independente do diretório que se esteje no momento.

    4. O que é e para que serve uma IDE? As IDE são softwares que fornecem diversos recursos para facilitar o desenvolvimento do código fonte de uma determinada linguagem de programação. Alguns frameworks fornecem suporte para mais de uma linguagem de programação. Estes recursos podem ser configurados com frameworks especificos de cada linguagem ou da própria IDE. Facilita na organização dos códigos, na independência da aplicação com o sistema operacional, entre outros recursos especificos de cada IDE.

    5. Como se chama o arquivo de configuração de uma aplicação App Engine? Este arquivo se chama app.yaml .

    6. Qual o domínio padrão para acessar aplicações App Engine na internet? O dominio padrão é www.nomeDaAplicacao.appspot.com.br.

    ## Capítulo II 1. Qual o nome do arquivo de configuração do Google App Engine? Este arquivo se chama app.yaml .

    1. Para que serve o item application do arquivo de configuração? Para identificar a aplicação.

    2. Para que serve o item version do arquivo de configuração? É utilizada para verificar qual é a versão (version) da aplicação quando a aplicação for publicada.

    3. Qual endereço deve ser utilizado para acessar uma aplicação com id foo e versão 35? http://35-dot-foo.appspot.com.br

    4. Para que serve a seção libraries do arquivo de configuração? É utilizada configurar as bibliotecas que vão ser utilizadas na aplicação.

    5. Para que serve a seção handlers do arquivo de configuração? A seção handlers serve para mapear os scripts Python que serão executados de acordo com o path das requisições HTTP.

    6. Como são definidos os paths mapeados no arquivo de configuração? Os paths são mapeados por expressões regulares.

    7. Por que é necessário mapear RequestHandlers nos scripts Python? É necessário mapear RequestHandlers nos scripts Python porque é comum a definição de múltiplas classes em um arquivo. Sendo assim somente o mapeamento via arquivo de configuração não é suficiente para definir qual classe será utilizada para processar uma requisição.

    8. Para que serve a classe RequestHandler? Utilizada para definir o código que irá processar uma requisição HTTP.

    9. Como se relacionam os métodos da classe RequestHandler e os do protocolo HTTP? Os métodos da classe RequestHandler devem ser sobrescritos para processar as chamadas HTTP de tipo com mesmo nome. Por exemplo, uma chamada HTTP do tipo GET será processada no método sobrescrito get, Uma chamada do tipo POST em um método post e assim or diante.

    10. Para que serve o objeto Request? Serve como uma interface para acesso às informações sobre uma requisição HTTP.

    11. Como se obtém os valores de parâmetros enviados via query string em uma chamada HTTP do tipo GET? Esses valores são obtidos utilizando-se o método get do objeto Request.

    12. Para que serve o objeto Response? O objeto Response server como interface para construção de uma resposta HTTP. Ela contém os dados a serem enviados como resposta a uma requisição.

    13. Qual o método do objeto Response serve para enviar strings? O método write da classe Response serve para enviar strings.

    14. Como é possível enviar uma resposta para redirecionamento? Deve ser utilizado o método redirect do objeto RequestHandler, fornecendo como parâmetro o path desejado, no caso de redirecionamento para a mesmo aplicação, ou a url completa em caso contrário.

    ## Capítulo III

    1. Para que serve o Virtualenv? Serve para isolar nosso ambiente de desenvolvimento de um determinado projeto, assim garantimos que um pacote instalado não interfira nos demais que projetos que rodam na sua máquina.

    2. Qual a função do arquivo convention.py? O arquivo convention.py serve para fazer a ligação entre o Tekton e o Wepapp2.

    3. Por que é necessário incluir bibliotecas através de código no arquivo convention.py? É necessário incluir bibliotecas através de código no arquivo convention.py porque diferente de servidores tradicionais, na plataforma GAE não se tem acesso ao Sistema Operacional para se poder instalar as bibliotecas utilizando virtualenv e pip.

    4. Como ficaria a declaração de uma função para tratar a execução de chamada no path /usuario/salvar?nome=Renzo&idade=31? A declaração da função seria def salvar(nome,idade) e deveria constar no script usuario.py.

    5. Como se diferenciam parâmetros recebidos por injeção de dependência dos recebidos via requisição HTTP? Parâmetros recebidos via injeção de dependência possuem a convenção de conter o prefixo “_” antes de seus nomes. Parâmetros recebidos via requisição HTTP são escritos sem esse prefixo.

    6. Qual deve ser a posição de parâmetros recebidos via injeção de dependência? Parâmetros recebidos via injeção de dependência devem sempre ser os primeiros a serem declarados em uma função.

    7. Qual deve ser o parâmetro declarado quando for necessário fazer um redirecionamento? Quando for necessário fazer um redirecionamento deve ser declarada a dependência _handler para acessar a instância de RequestHandler que está tratando a requisição. Deve ser utilizado o método redirect desse objeto para se efetuar o redirecionamento.

    8. Qual o script e função devem ser utilizados para se calcular paths com base em uma função? Deve ser utilizada a função to_path do módulo tekton.router para se calcular o path respectivo a uma função, que deve ser passada como parâmetro.
